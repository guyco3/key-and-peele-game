# Contributing to Key & Peele Game

Thank you for considering contributing to this project! Please follow these guidelines to help us keep a high-quality codebase and a welcoming community.

## Conventional Commits

We use [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) for all commit messages. This helps automate releases, changelogs, and makes it easier to understand the history of the project.

**Examples:**
- `feat: add new sketch search UI`
- `fix: correct scoring bug in backend`
- `docs: update README with setup instructions`
- `refactor: simplify socket event handling`

See the [Conventional Commits documentation](https://www.conventionalcommits.org/en/v1.0.0/) for full details and allowed types.

## Pull Requests

- Fork the repository and create your branch from `main`.
- Use clear, descriptive branch names (e.g., `feat/autocomplete-search`, `fix/round-timing`)
- Ensure your code passes all tests and lints before submitting a PR.
- Add or update documentation as needed.
- Reference related issues in your PR description if applicable.
- Be responsive to code review feedback.

## Code Style

- Use the existing formatting and linting rules in the project.
- Write clear, concise code and comments.
- Prefer small, focused PRs over large, sweeping changes.

## Issues

- Search existing issues before opening a new one.
- Provide as much detail as possible (steps to reproduce, screenshots, etc.)

## Community

- Be respectful and inclusive in all interactions.
- Follow the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).

---

Thank you for helping make this project awesome!
